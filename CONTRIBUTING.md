# 기여 지침

## 작업 절차

1. `main` 브랜치에서 최신 내용을 가져옵니다.
2. 기능 또는 수정 목적에 맞는 별도 브랜치를 만듭니다.
3. 변경 사항을 작고 명확한 커밋으로 나눕니다.
4. 로컬 빌드와 테스트를 확인합니다.
5. Pull Request를 생성하고 변경 목적과 검증 내용을 작성합니다.

## 커밋 메시지 권장 형식

- `feat:` 기능 추가
- `fix:` 버그 수정
- `docs:` 문서 변경
- `refactor:` 동작 변경 없는 구조 개선
- `test:` 테스트 추가 또는 수정
- `chore:` 설정 및 유지보수 작업

## 주의 사항

- 비밀번호, API 키, 토큰, 인증서, 개인정보를 저장소에 올리지 않습니다.
- 생성 파일, 빌드 산출물, 로그 파일은 커밋하지 않습니다.
- 동작 변경에는 가능한 범위에서 테스트 또는 검증 절차를 포함합니다.

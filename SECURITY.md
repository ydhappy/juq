# 보안 정책

## 보안 문제 보고

공개 Issue에는 비밀번호, 토큰, 개인정보, 취약점 재현용 비밀정보를 작성하지 마세요.

저장소 소유자에게 비공개 방식으로 다음 내용을 전달해 주세요.

- 영향받는 파일 또는 기능
- 재현 조건과 절차
- 예상 영향 범위
- 가능한 완화 방법

## 지원 범위

현재 개발 중인 `main` 브랜치의 최신 상태를 기준으로 보안 수정을 검토합니다.

## 비밀정보 관리

- `.env`와 로컬 인증 파일은 커밋하지 않습니다.
- 노출된 토큰이나 비밀번호는 파일 삭제만으로 처리하지 않고 즉시 폐기·재발급합니다.
- 예제 설정에는 실제 값 대신 자리표시자를 사용합니다.
